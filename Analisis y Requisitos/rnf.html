Requisitos No Funcionales

Facilidad de Uso

ID Requisito Descripción
RNF-001 Publicidad Antes de intensificar el uso del sistema, los usuarios deben conocer su modo de uso.
RNF-002 Actualización de Información Actualizar la información por ingreso o retiro de algún trabajador.
RNF-003 Diseño Intuitivo Diseño adecuado a las necesidades del usuario, interfaz atractiva y carga rápida.

Confiabilidad

ID Requisito Descripción
RNF-004 Disponibilidad El sistema debe estar disponible las 24 horas del día.
RNF-005 Actualización de BD Asegurar la permanente actualización de la base de datos al registrar información.

Ambiente de Trabajo

ID Requisito Descripción
RNF-006 Tiempo de Respuesta Minimizar el tiempo de respuesta a un promedio de 15 segundos.
RNF-007 Capacidad de Almacenamiento Asignar suficiente espacio a la base de datos para soportar grandes cantidades de información.
RNF-008 Configuración del Equipo Configurar adecuadamente el equipo para soportar la instalación del sistema.

Restricciones de Diseño

ID Requisito Descripción
RNF-009 Lenguaje de Programación El sistema debe implementarse en Java.
RNF-010 Licenciamiento El sistema requiere licenciamiento para su desarrollo.

Seguridad

ID Requisito Descripción
RNF-011 Encriptación de Contraseñas Las contraseñas deben ser almacenadas encriptadas con un algoritmo criptográfico.
RNF-012 Backup de Datos Realizar un backup de datos en disco o cinta para disponer de un nivel adicional de protección.
RNF-013 Operaciones Seguras Todas las operaciones de consulta y modificación deben ser seguras.

Documentación y Sistemas de Ayuda

ID Requisito Descripción
RNF-014 Capacitación Capacitación a los usuarios para asegurar un buen manejo del sistema.
RNF-015 Manuales Proveer manuales de usuario.

Interfaz de Usuario

ID Requisito Descripción
RNF-016 Dispositivo El sistema será manejado en un computador de escritorio.
RNF-017 Colores Los colores de la aplicación serán gris oscuro, azul claro y blanco.
RNF-018 Fuente El texto será en color negro, con fuente Arial tamaño 11.

Interfaces de Comunicación

ID Requisito Descripción
RNF-019 Acceso Web El acceso al software será vía web, asegurando estabilidad y seguridad.
RNF-020 Comunicación Estable Debe existir comunicación estable con la interfaz de usuario.
