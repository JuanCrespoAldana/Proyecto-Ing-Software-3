FURPS vs ISO 25010

Modelo FURPS

El modelo FURPS (Functionality, Usability, Reliability, Performance y Supportability) fue desarrollado por Hewlett-Packard y sirve para clasificar los requisitos de software en categorías funcionales y no funcionales. Este modelo ayuda a establecer criterios de calidad centrados en el rendimiento, la usabilidad, la confiabilidad y la capacidad de soporte del sistema.

Norma ISO/IEC 25010

La norma ISO/IEC 25010 define un modelo de calidad del software compuesto por características y subcaracterísticas que permiten evaluar la calidad de un producto o sistema. Incluye aspectos como la seguridad, la mantenibilidad, la portabilidad y la compatibilidad, con el objetivo de garantizar que el software cumpla con las necesidades del usuario y los estándares internacionales.
