Diagrama de Clases

El siguiente diagrama muestra las principales clases del sistema para la detección de deshidratación, incluyendo usuarios, sensores, reportes y notificaciones.

@startuml
class Usuario {
  - idUsuario: int
  - nombre: String
  - correo: String
  - edad: int
  - contraseña: String
  + iniciarSesion()
  + registrar()
}

class Familiar {
  - idFamiliar: int
  - nombre: String
  - parentesco: String
}

class Administrador {
  - idAdmin: int
  - nombre: String
  + gestionarUsuarios()
  + configurarParametros()
}

class Dispositivo {
  - idDispositivo: int
  - modelo: String
  + conectar()
}

class Sensor {
  - idSensor: int
  - tipo: String
  - valor: double
  + obtenerLectura()
}

class Reporte {
  - idReporte: int
  - fecha: Date
  - nivelRiesgo: String
}

class Notificacion {
  - idNotificacion: int
  - mensaje: String
  + enviar()
}
